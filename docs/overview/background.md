# 需求背景

## 问题

推荐系统需要持续、稳定、多样化的内容供给。传统做法依赖人工编辑或单一数据源，面临以下挑战：

### 1. 内容来源单一

- 仅依赖少数 RSS 源或手动录入
- 无法覆盖热点、长尾内容
- 缺乏国际/国内多平台覆盖

### 2. 内容质量参差不齐

- 原始内容包含 HTML 残留、广告、重复段落
- 缺乏统一的质量评分体系
- 无去重机制，相同内容重复入库

### 3. 版权风险

- 直接使用原文存在版权问题
- 需要对内容进行改写、摘要或伪原创处理

### 4. 内容池管理困难

- 无法自动清理过期、低质量内容
- 缺乏容量控制，数据库无限膨胀
- 清理操作缺乏审核机制，误删风险高

### 5. 与推荐系统耦合

- 内容采集逻辑嵌入推荐系统中
- 无法独立迭代、部署、扩展

## 解决方案

Content Supply Platform 作为一个**独立的内容供给中间件**，解决上述所有问题：

```
外部内容源 → Content Supply Platform → 共享存储 → 推荐系统
  (RSS/网页/热搜)  (采集/处理/改写/管理)  (MySQL+Redis)  (召回/排序)
```

### 核心价值

| 价值 | 说明 |
|------|------|
| 多源采集 | RSS、通用网页、国内外热搜词，覆盖面广 |
| 智能处理 | 自动去重、标签提取、质量评分 |
| LLM 改写 | 多模式改写，规避版权风险 |
| 统一管理 | 单池多标签，灵活路由 |
| 安全清理 | 审核制清理，防止误删 |
| 独立部署 | 与推荐系统解耦，独立迭代 |

## 目标用户

- **推荐系统工程师** — 需要稳定的内容供给管道
- **内容运营** — 需要管理内容源、监控内容质量
- **平台管理员** — 需要配置清理策略、审核清理操作
