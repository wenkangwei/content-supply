> 版本: v1.0 | 日期: 2026-05-08 | 状态: 初稿

## 一、项目背景

### 1.1 项目定位

为 Content Supply Platform（后端）构建前端管理界面，定位为**内容展示为主、管理为辅**的单页面应用。风格参考 [AIHOT](https://aihot.virxact.com/) 的暗色主题内容流。

### 1.2 核心目标

| 目标 | 说明 |
|------|------|
| 内容展示 | 以卡片流/列表形式展示采集到的内容，支持搜索、筛选、排序 |
| Feed 管理 | CRUD 操作 RSS/Atom 订阅源 |
| 爬取控制 | 手动触发爬取、查看任务状态 |
| 热搜监控 | 查看各平台热搜词、触发热搜采集 |
| 内容改写 | LLM 改写操作（单条/批量） |
| 清理管理 | 清理策略查看、审核操作 |

### 1.3 目标用户

- **内容运营** — 浏览/搜索/筛选内容，管理订阅源
- **平台管理员** — 监控爬取状态，审核清理操作，配置策略
- **开发者** — 调试 API，查看系统健康状态

### 3.3 关键技术决策

#### 3.3.1 为什么选 React + Vite + TailwindCSS

| 对比项 | React+Vite+TW | Vue3+ElementPlus | Next.js+TW |
|--------|--------------|------------------|------------|
| 暗色主题 | TW 原生支持 | 需自定义主题 | TW 原生支持 |
| 构建速度 | 极快（Vite） | 快（Vite） | 较慢（Webpack） |
| 灵活度 | 高 | 中（Element约束） | 高 |
| 复杂度 | 中 | 低 | 高 |
| 适配AIHOT风格 | 容易定制 | 需大量覆盖样式 | 容易但过重 |

#### 3.3.2 暗色主题方案

```css
/* TailwindCSS dark mode via class strategy */
:root {
  --bg-primary: #060814;      /* AIHOT 深蓝黑 */
  --bg-secondary: #0f1729;    /* 卡片背景 */
  --bg-tertiary: #1a2332;     /* hover/active */
  --text-primary: #e2e8f0;    /* 主文字 */
  --text-secondary: #94a3b8;  /* 辅助文字 */
  --accent-blue: #3b82f6;     /* 主强调色 */
  --accent-green: #22c55e;    /* 成功/在线 */
  --accent-red: #ef4444;      /* 错误/删除 */
  --accent-yellow: #eab308;   /* 警告/热搜 */
  --border: #1e293b;          /* 边框 */
}
```

#### 3.3.3 API 代理配置

Vite 开发时通过 `proxy` 转发 API 请求到后端：


### 4.1 API 通信规范

| 项目 | 规范 |
|------|------|
| 协议 | HTTP/1.1, JSON |
| Base URL | `/` (Vite proxy 转发) |

## 五、开发规范

详见 [v1.0-开发规范.md](./v1.0-开发规范.md)

### 5.1 核心原则

1. **组件化开发**: 页面由小组件组合而成，单一职责
2. **类型安全**: 所有 API 数据、Props、State 均有 TypeScript 类型
3. **暗色优先**: 默认暗色主题，通过 CSS 变量支持主题切换
4. **响应式**: 支持桌面和平板，移动端基础适配
5. **无障碍**: 语义化 HTML，键盘导航支持

### 5.2 命名规范

| 类型 | 规范 | 示例 |
|------|------|------|
| 组件文件 | PascalCase | `ContentCard.tsx` |
| 工具文件 | camelCase | `formatDate.ts` |
| 样式类 | Tailwind 原子类 | `bg-slate-900 text-slate-100` |
| 状态 Store | use前缀 | `useFeedStore` |
| API 函数 | 动词+名词 | `getFeeds`, `createFeed` |
| 自定义 Hook | use前缀 | `usePagination` |
