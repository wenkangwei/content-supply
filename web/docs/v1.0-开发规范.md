# Content Supply Web - v1.0 开发规范

> 版本: v1.0 | 日期: 2026-05-08 | 强制规范，所有开发必须遵守

---

## 一、代码风格

### 1.1 TypeScript

- **严格模式**: `strict: true`，禁止 `any`，使用 `unknown` 替代
- **类型导出**: 每个 API 类型文件独立导出，不使用 `export *`
- **接口优先**: 优先用 `interface`，复杂联合类型用 `type`
- **枚举**: 使用 union string literal 而非 `enum`（`'active' | 'paused'`）

### 1.2 命名

| 类型 | 规范 | 示例 |
|------|------|------|
| 组件 | PascalCase | `ContentCard.tsx` |
| Hook | use + PascalCase | `usePagination.ts` |
| Store | use + PascalCase + Store | `useFeedStore.ts` |
| API 文件 | camelCase | `feeds.ts`, `items.ts` |
| API 函数 | 动词 + 名词 | `getFeeds()`, `createFeed()` |
| 常量 | UPPER_SNAKE | `API_BASE_URL` |
| CSS 类 | Tailwind 原子类 | `bg-slate-900` |
| 事件处理 | handle + 动作 | `handleClick`, `handleSubmit` |
| 布尔变量 | is/has/can 前缀 | `isLoading`, `hasError` |

### 1.3 文件组织

```
一个组件一个文件
文件名 = 默认导出的组件名
index.ts 仅用于 re-export，不包含逻辑
```

### 1.4 Import 顺序

```typescript
// 1. React/第三方库
import { useState, useEffect } from 'react';
import { useNavigate } from 'react-router-dom';

// 2. 内部模块（按层级）
import { getItems, Item } from '@/api/items';
import { useItemStore } from '@/stores/useItemStore';
import { formatDate } from '@/utils/format';

// 3. 组件
import { Button } from '@/components/ui/Button';
import { ContentCard } from '@/components/shared/ContentCard';
```

## 二、组件规范

### 2.1 组件结构

```typescript
// 组件模板
interface Props {
  title: string;
  onAction?: () => void;
}

export function MyComponent({ title, onAction }: Props) {
  // 1. hooks
  const [state, setState] = useState(false);

  // 2. effects
  useEffect(() => { ... }, []);

  // 3. handlers
  const handleClick = () => { ... };

  // 4. render
  return (
    <div className="...">
      ...
    </div>
  );
}
```

### 2.2 TailwindCSS 使用规范

- **禁止内联 style**: 一律使用 Tailwind 类
- **响应式前缀**: `md:` 作为桌面断点，不使用 `lg:`/`xl:`
- **暗色前缀**: 使用 CSS 变量 + Tailwind 自定义颜色，不用 `dark:` 前缀（默认暗色）
- **类名顺序**: 布局 → 尺寸 → 间距 → 排版 → 背景 → 边框 → 效果
- **重复组合**: 超过3次使用的类组合提取为 `@apply` 或组件

### 2.3 暗色主题设计规范

```
背景层级:
  - 页面底色: bg-[#060814]
  - 侧边栏: bg-[#0c1021]
  - 卡片/面板: bg-[#0f1729]
  - hover: bg-[#1a2332]
  - active: bg-[#243044]

文字层级:
  - 主标题: text-slate-100
  - 正文: text-slate-200
  - 辅助: text-slate-400
  - 禁用: text-slate-500

强调色:
  - 主色: text-blue-400 / bg-blue-500
  - 成功: text-green-400 / bg-green-500
  - 警告: text-yellow-400 / bg-yellow-500
  - 错误: text-red-400 / bg-red-500
```

## 三、API 调用规范

### 3.1 Axios 实例配置

```typescript
const apiClient = axios.create({
  baseURL: '/',  // Vite proxy handles routing
  timeout: 30000,
  headers: { 'Content-Type': 'application/json' },
});
```

### 3.2 请求/响应拦截

- **请求**: 无需附加 token（本地部署无认证）
- **响应**: 统一解包 `response.data`
- **错误**: 统一 Toast 提示，404 静默处理

### 3.3 Loading 状态

每个 API 调用必须维护 loading 状态:

```typescript
const [loading, setLoading] = useState(false);

const fetchData = async () => {
  setLoading(true);
  try {
    const data = await getItems(params);
    setItems(data.items);
  } catch (err) {
    // Toast error
  } finally {
    setLoading(false);
  }
};
```

## 七、构建与部署

### 7.1 开发

```bash
cd web
npm install
npm run dev        # http://localhost:5173
```

### 7.2 构建

```bash
npm run build      # 输出到 dist/
npm run preview    # 预览构建结果
```

### 7.3 生产部署

构建产物为纯静态文件，可部署到 Nginx / CDN / GitHub Pages:
- Nginx 需配置 `try_files $uri $uri/ /index.html`（SPA路由）
- API 请求需反向代理到 `localhost:8010`
